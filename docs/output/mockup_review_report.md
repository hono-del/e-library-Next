## 5. UX・導線レビュー

### ユーザーの開始状況
- 整備士が車両の診断を開始しようとしている
- DTCを読み取り、原因を調べたい
- マニュアルや事例を探す時間を削減したい

### ユーザーの目的
- 素早く診断・修理に必要な情報を得る
- 作業中に迷わず次のステップに進む
- 安全に作業を完了する

### 主要な行動フロー
1. トップページで「作業を開始する」をクリック
2. 車両情報（車種、年式、症状、DTC）を入力
3. 作業セッション画面で推薦情報を確認
4. 必要に応じて詳細情報を確認
5. 工程を進めながら作業
6. 作業完了

### 迷いやすいポイント
❌ **トップページ**: このサービスが何をするのか、初見で分からない  
❌ **作業セッション画面**: なぜこの情報が表示されているのか分からない  
❌ **工程変更**: 次の工程に進むと何が変わるのか分からない  
❌ **アクションバー**: 「追加情報を探す」と「質問を投稿」の違いが不明

### 操作負荷が高いポイント
⚠️ 工程変更が手動のみ（自動判定がない）  
⚠️ DTC入力が手入力のみ（診断機連携がない）

### 次アクションが不明確なポイント
❌ AIアシスト要約を読んだ後、何をすべきか示されていない  
❌ マニュアルを見た後、作業に戻る導線が不明確

### より自然な導線案
1. **トップページ**: 価値提案 → 使い方説明 → 作業開始
2. **作業セッション**: 状況説明 → AI推薦理由 → 推薦情報 → 次アクション明示
3. **工程変更**: 変更前に「次の工程では〇〇の情報を表示します」と予告

## 6. デモストーリーレビュー

### デモで伝わる価値
✅ 作業セッションを開始すると、情報が自動で表示される  
✅ AIが要約を提示する  
✅ 関連するマニュアル・TIE・問題交流が一覧表示される

### デモで伝わりにくい価値
❌ **「探すをなくす」という本質**: 従来の検索が不要になることが見えない  
❌ **時間削減効果**: 10-20分 → 2-4分の削減が体感できない  
❌ **工程別の変化**: 工程ごとに情報が変わることが1回の操作では分からない  
❌ **メーカー側の価値**: 利用ログ・頻出課題の可視化が見えない

### Before/Afterが弱い部分
❌ **従来**: マニュアルサイトで検索 → 複数ページを見る → 関連情報を探す（10-20分）  
❌ **新サービス**: 車両情報入力 → 必要な情報が自動表示（2-4分）  
→ **改善案**: トップページに従来の検索画面のスクリーンショットと比較表示

### 山場が不足している部分
現状は「情報が表示される」で終わっており、感動ポイントがない。

**改善案**:
1. 工程変更ボタンを押すと、画面が切り替わり「作業計画に必要な部品リスト・工数見積もりが自動表示される」という驚きを演出
2. AIアシストが「P0420の95%はO2センサー交換で解決しています」と具体的な成功率を示す

### ステークホルダーが疑問に思いそうな部分
❓ 「AIはどこまで正確なのか？」→ 免責事項・参照元リンクが必要  
❓ 「既存のマニュアルとどう違うのか？」→ Before/After比較が必要  
❓ 「工程別とは具体的にどう変わるのか？」→ 工程変更デモが必要  
❓ 「導入コストは？」→ （今回のモックアップでは不要）

### デモ順序の改善案

**現状**: トップ → フォーム → セッション詳細  
**改善案**:

```
1. 【Before画面】従来の情報探索（10-20分）
   - マニュアルサイトで検索
   - 複数ページを行き来
   - 関連情報が見つからない

2. 【After: トップページ】e-library Nextの価値提案
   - 「作業に必要な情報が自動で届く」
   - 「探す時間を80%削減」

3. 【フォーム】車両情報入力（30秒）

4. 【セッション詳細: 診断工程】
   - AIアシスト: P0420の診断手順を要約
   - マニュアル: 診断フローチャート
   - TIE事例: 類似症状の対応事例
   - 注意事項: 作業前の安全確認
   - 所要時間: 2-4分

5. 【工程変更デモ】診断 → 作業計画
   - 画面が切り替わり、部品リスト・工数見積もりが表示される
   - 「工程ごとに必要な情報が自動で変わります」

6. 【メーカー視点】分析ダッシュボード（別画面）
   - 頻出課題の可視化
   - 自動提示の活用率
```
